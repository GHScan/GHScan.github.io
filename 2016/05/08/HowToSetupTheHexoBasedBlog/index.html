怎么用GithubPages+Hexo+NexT搭建博客

发表于 2016-05-08   |   分类于 Engineering   |  

我想自己搭博客很久了，之前是不会 Web，等到终于会做网站的时候，已经嫌麻烦不太想弄了，所以转而寻求现成的博客生成、托管方案。

考虑到，博客托管和文章备份一定要靠谱，一番考察，还是 GitHub Pages 最省事；而基于 GitHub Pages 的静态网站方案中，对比Jekyll 和 Hexo ，后者诞生得晚、活跃的主题/插件多，安装和部署都更简单。在 Hexo 主题中，NexT (Demo)又比绝大部分竞争对手更成熟，于是最终方案就是 GitHub Pages + Hexo + NexT.

在考察各种教程和主题后，这是我的实施过程：

  1. 先看GitHub Pages文档，用它托管一个只包含 index.html 的静态网站。成功后，你能通过http://username.github.io访问这个静态博客。
  2. 注册域名，添加 CNAME record 重定向到你刚创建的 GitHub Pages 上的静态博客。另外，GitHub Pages 那边也需要配置一下： Using a custom domain with GitHub Pages。
  3. 了解 Hexo 这个静态博客生成器，按照 Doc 在本机创建一个博客模板。此时运行hexo generate命令可以根据初始模板生成对应的静态网站，再 push 到前面创建的 GitHub Pages 的 Repo 的话，其实一个基本的博客已经完工。这里有篇很详细的教程：史上最详细的Hexo博客搭建图文教程。
  4. 上面的博客还太简单，所以我们要用 Hexo 主题。Github 上 Star 最多的是NextT，质量很高，也挺好看 (Demo)，改几行配置文件就能完成很多事情：
    • 多说/Disqus 评论
    • 百度/腾讯/Google 统计
    • 微信打赏/公众号

至此全部完工，以后想添加新文章，在 Hexo 的模板 Repo 里hexo new "post_name"，再运行hexo generate，最后 push 到https://github.com/username/username.github.io ，一篇新的博文就发布了。
